Empirical Runtime and Memory Comparison

Here are some initial runtime and memory benchmarks. All of the algorithms are ran in exactly the setting that the experiments in the paper were done with. You can see the runtimes for each algorithm as well as the peak allocated memory that the algorithm used.

The runtime is comparable. However, the space requirement for M2UCB seems to be much higher than all the other algorithms. I think this has to do both with model selection as well as the use of caching to speed up the algorithm. So the use of memory vs. runtime is a tradeoff that we discuss in more detail in our next revision.
